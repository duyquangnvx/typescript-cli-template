# Data Model: Build TypeScript CLI Template

**Date**: 2026-02-19
**Feature**: 001-build-cli-template

## Overview

This feature has no persistent data storage. The data model describes
the runtime entities that structure the CLI application.

## Entities

### Command

Represents a CLI subcommand registered with the Commander.js program.

| Attribute   | Type                 | Description                                   |
| ----------- | -------------------- | --------------------------------------------- |
| name        | string               | Subcommand name used on the command line      |
| description | string               | One-line description shown in --help output   |
| arguments   | Argument[]           | Positional arguments the command accepts      |
| options     | Option[]             | Named options (flags) the command accepts     |
| handler     | (args, opts) => void | Function invoked when the command is executed |

**Lifecycle**: Commands are registered at program startup (before
`.parse()`) and remain immutable for the lifetime of the process.

**Uniqueness**: Command names MUST be unique within the program.
Commander.js enforces this at registration time.

### Argument

Represents a positional argument on a command.

| Attribute   | Type    | Description                              |
| ----------- | ------- | ---------------------------------------- |
| name        | string  | Argument name shown in usage synopsis    |
| required    | boolean | Whether the argument must be provided    |
| description | string  | Help text for the argument               |
| default     | any     | Default value if not provided (optional) |

### Option

Represents a named option (flag) on a command.

| Attribute   | Type    | Description                                             |
| ----------- | ------- | ------------------------------------------------------- |
| flags       | string  | Short and long flag syntax (e.g., "-n, --name <value>") |
| description | string  | Help text for the option                                |
| required    | boolean | Whether the option must be provided                     |
| default     | any     | Default value if not provided (optional)                |

### CLI Application

The top-level program instance that owns all commands.

| Attribute   | Type      | Description                         |
| ----------- | --------- | ----------------------------------- |
| name        | string    | Program name (shown in help/usage)  |
| version     | string    | Semantic version from package.json  |
| description | string    | Program description shown in --help |
| commands    | Command[] | Registered subcommands              |

**Global flags** handled by the CLI Application:

- `--help` — auto-generated by Commander.js
- `--version` — set via `.version()` from package.json
- `--json` — custom global option toggling JSON output mode

## Relationships

```text
CLI Application 1──* Command
Command 1──* Argument
Command 1──* Option
```

## State Transitions

Not applicable. The CLI is stateless — it receives input, processes
a single command, produces output, and exits.
